# 회원 도메인 원칙 (DOMAIN-MEMBER-CONSTITUTION)

## 1. 개요

시냅스의 회원 도메인은 사용자 인증과 신원 관리를 담당한다.

## 2. 핵심 원칙

- **인증 위임**: 회원 인증은 Supabase Auth에 전적으로 위임한다. 자체 인증 로직을 구현하지 않는다.
- **세션 쿠키 기반**: 세션은 쿠키로 관리한다. localStorage/sessionStorage 기반 토큰 저장을 사용하지 않는다.
- **라우트 보호**: 인증이 필요한 모든 라우트는 서버 단에서 검증한다. 클라이언트 단 보호만으로는 충분하지 않다.
- **데이터 격리**: 모든 사용자 데이터는 RLS(Row Level Security)로 격리한다. 한 사용자가 다른 사용자의 데이터에 접근할 수 없다.
