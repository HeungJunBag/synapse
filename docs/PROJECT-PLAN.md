## 1. 프로젝트 개요

### 1-1. 한 줄 소개

> 생각과 생각을 연결하는 제텔카스텐 기반 개인 지식 관리 도구

### 1-2. 문제 정의

기존 노트 앱(Notion, Bear 등)은 정보를 **폴더/계층 구조**로 저장한다. 이 방식은 아이디어 간의 **연결과 관계**를 표현하지 못한다. 생각은 선형이 아니라 그물망 구조로 발전한다.

### 1-3. 솔루션

제텔카스텐(Zettelkasten) 방법론을 적용한다.
- **원자성**: 노트 하나에 생각 하나만 담는다
- **연결성**: `[[노트 제목]]` 형태로 노트 간 링크를 걸어 지식을 그물망으로 엮는다
- **시각화**: 연결 관계를 그래프로 시각화해 전체 지식 지형을 한눈에 파악한다

## 3. 핵심 기능

### 현재 구현된 기능 (v0.1)

| 기능 | 설명 |
|------|------|
| 회원가입 / 로그인 | 이메일 + 패스워드, Supabase Auth |
| 노트 CRUD | 생성, 편집(TipTap 리치 에디터), 삭제 |
| `[[링크]]` 자동완성 | 에디터에서 `[[` 입력 시 기존 노트 목록 드롭다운 |
| 링크 자동 동기화 | 저장 시 `[[링크]]` 파싱 → `note_links` 테이블 자동 갱신 |
| 아웃고잉 링크 패널 | 현재 노트에서 연결된 노트 목록 표시 |
| 백링크 패널 | 현재 노트를 참조하는 노트 목록 표시 |
| 태그 표시 | 노트에 연결된 태그 칩 표시 |
| 그래프 뷰 | 노트 간 연결 관계를 force-directed 그래프로 시각화 |
| 그래프 → 노트 이동 | 그래프 노드 클릭 시 해당 노트로 이동 |

### 다음 단계 후보 기능 (미구현)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 노트 검색 | 제목/본문 전체 텍스트 검색 | 높음 |
| 태그 필터링 | 태그로 노트 목록 필터 | 중간 |
| AI 자동 태그 생성 | 노트 저장 시 내용 기반 태그 자동 추천 | 중간 |
| 모바일 반응형 | 스마트폰에서도 편하게 사용 | 중간 |
| 노트 공유 | 특정 노트를 링크로 공유 | 낮음 |
| 마크다운 내보내기 | 노트를 .md 파일로 다운로드 | 낮음 |

## 5. 데이터 구조 (요약)

```
notes          — 노트 본문 (user_id, title, content, timestamps)
note_links     — 노트 간 단방향 링크 (source_note_id → target_note_id)
tags           — 태그 마스터 (user_id, name)
note_tags      — 노트-태그 다대다 조인
```

모든 테이블에 RLS 적용. 사용자는 자신의 데이터에만 접근 가능.

## 6. 화면 구성

```
[로그인 / 회원가입]
  └─ 다크 모드 UI

[메인 앱] — 탭 전환
  ├─ 📝 노트 탭
  │    ├─ 좌측: 노트 목록 (+ 새 노트 버튼)
  │    └─ 우측: TipTap 에디터
  │              ├─ 제목 입력
  │              ├─ 본문 (리치 텍스트 + [[링크]] 자동완성)
  │              ├─ 저장 / 삭제 버튼
  │              └─ 하단: 아웃고잉 링크 + 백링크 + 태그 칩
  └─ 🕸 그래프 탭
       └─ force-directed 노드 그래프 (클릭 → 노트 이동)
```
