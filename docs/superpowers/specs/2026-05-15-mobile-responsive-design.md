# 모바일 반응형 디자인 스펙

**날짜:** 2026-05-15  
**범위:** NoteLayout.tsx, NoteList.tsx  
**브레이크포인트:** Tailwind `md` (768px)

---

## 목표

모바일(768px 미만) 화면에서 시냅스 앱이 정상적으로 사용 가능하도록 반응형 레이아웃을 구현한다. 데스크탑 레이아웃(사이드바 + 에디터 좌우 분할)은 변경하지 않는다.

---

## 승인된 설계

### 레이아웃: 슬라이드 드로어 방식

- 모바일에서 사이드바(NoteList)는 기본적으로 숨겨져 있다.
- ☰ 버튼을 누르면 왼쪽에서 슬라이드인 되며 에디터 위에 오버레이로 표시된다.
- 드로어 바깥 영역(반투명 backdrop)을 클릭하면 드로어가 닫힌다.
- 드로어에는 기존 NoteList 전체(새 노트 버튼 + 검색 + 태그 필터 + 노트 목록)가 표시된다.

### 헤더 구조 (모바일)

**1행 (항상 보임):**
- 왼쪽: ☰ 햄버거 버튼 (모바일만 표시)
- 중앙: 시냅스 로고 텍스트
- 오른쪽: 로그아웃 버튼

**2행 (헤더 아래 별도 줄):**
- 📝 노트 탭 | 🕸 그래프 탭 (전체 너비, 두 탭이 나란히)

**데스크탑 (md 이상):**
- 기존 구조 유지: 왼쪽=시냅스, 중앙=탭 버튼들, 오른쪽=이메일+로그아웃

---

## 컴포넌트별 변경사항

### NoteLayout.tsx

1. **상태 추가:**
   - `drawerOpen: boolean` — 모바일 드로어 열림/닫힘 상태

2. **헤더 재구조화:**
   - 모바일: `☰ | 시냅스 | 로그아웃` (1행) + 탭바 (2행)
   - 데스크탑: 기존 헤더 구조 유지 (`md:flex` 클래스로 조건부 표시)
   - 햄버거 버튼: `md:hidden` (모바일만 표시)
   - 이메일 주소: `hidden md:block` (데스크탑만 표시)

3. **드로어 오버레이:**
   - `drawerOpen` 시 왼쪽에서 슬라이드인 되는 fixed 패널
   - 뒤에 반투명 backdrop (`bg-black/40`) — 클릭하면 드로어 닫힘
   - 드로어 너비: `w-64` (256px)

4. **NoteList 렌더링:**
   - 모바일: 드로어 안에서만 표시 (`md:hidden` wrapper)
   - 데스크탑: 기존 사이드바 위치 (`hidden md:flex` wrapper)

5. **노트 선택 시 드로어 자동 닫힘:**
   - `selectNote` 호출 시 `setDrawerOpen(false)` 호출

### NoteList.tsx

- 드로어 안에서도 정상 동작하도록 별도 스타일 변경 없음
- 기존 `aside` 태그의 `w-60 flex-shrink-0` 클래스는 드로어 컨텍스트에서 override됨
- NoteList는 props/로직 변경 없이 레이아웃만 다른 위치에서 렌더링됨

---

## Tailwind 클래스 패턴

| 용도 | 클래스 |
|------|--------|
| 모바일만 표시 | `md:hidden` |
| 데스크탑만 표시 | `hidden md:block` or `hidden md:flex` |
| 드로어 슬라이드인 | `translate-x-0` (열림) / `-translate-x-full` (닫힘) |
| 백드롭 | `fixed inset-0 bg-black/40 z-30` |
| 드로어 패널 | `fixed inset-y-0 left-0 w-64 z-40 bg-slate-50 ...` |

---

## 접근성

- 햄버거 버튼: `aria-label="메뉴 열기"` / `aria-label="메뉴 닫기"` (drawerOpen 상태에 따라)
- 백드롭 버튼: `aria-label="메뉴 닫기"`

---

## 데스크탑 레이아웃 불변 보장

- 데스크탑에서 기존 동작이 깨지지 않아야 한다.
- `md:` prefix 클래스를 통해 768px 이상에서는 기존 레이아웃이 그대로 유지된다.
